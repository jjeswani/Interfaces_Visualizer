Interfaces Visualizer — How to Use
📂 Getting Started
📄Upload a CSV, XLSX or XLS file — one row per interface, with a header. Minimum 3 columns: Sender, Data Object, Receiver. Optionally add a Middleware column. Any other extra columns (Domain, Status, etc.) become Filters automatically.
🗂️Map your columns in the dropdowns (Sender, Interfaced Object, Receiver, Middleware), then click Build Network.
🔽 Filters & Context
🏷️Filters (teal) — Pre-filter by any extra column. Type or select a value → chip appears. Multiple filters use AND logic. Middleware column is excluded here (use the tiles instead).
🔎System 1 / System 2 — narrow to interfaces involving one or both systems; enters Focus mode. Dropdowns show in-scope values first when a context is active.
🔷Data / Interface Object — show all interfaces carrying a specific data object across all systems.
✕Click any active tag or ✕ to clear that filter.
🔗 Integration Middleware Tiles
🟦Shown below the Filters bar when a Middleware column is mapped. One tile per middleware value showing its total interface count. A w/o Middleware tile shows unmapped interfaces.
👆Click a tile — filters the diagram to that middleware only. Tile turns blue (active). Click again to deselect. All other tiles remain visible.
⊕Open Focus Mode button appears in overview when a middleware tile is active — click to enter the 3-column focus layout scoped to that middleware's interfaces.
🏷️Tiles update dynamically when any filter or context changes — counts always reflect the current scope.
🌐 Overview Mode
🖱️Hover a node to highlight its connections. Click to enter Focus mode with that system as anchor.
🏷️Click a link label — filters by that data object and enters Focus mode. Default label shows Middleware (if mapped); toggle to Object Names with the ⊙ button.
🔍Scroll to zoom · drag to pan · ⊙ resets to fit view.
⊞Show Interface Matrix — switch to a Sender × Receiver grid. Hover cells to highlight; click to enter Focus mode for that pair.
⚓ Focus Mode
⚓Anchor system (centre) is the hub. Left column = senders (amber/green). Right column = receivers (blue). Grey arcs connect non-anchor peer systems.
👆Click a system — pins highlight; click again to unpin. Click a link — drill into that pair's interfaces in detail.
⊙Show Object Names — switches link labels from Middleware names to Data Object names (and back).
↻Rotate 90° — rotates the canvas clockwise; click Default View to restore.
↕Drag a node vertically to reorder it within its column.
⎋Esc or double-click empty canvas to return to Overview.
🎨 Colour Coding
AmberSends TO the anchor
BlueReceives FROM the anchor
GreenBidirectional — sends and receives
Light blueAnchor (hub) system
GreyPeer connection between non-anchor systems
📤 Export (Focus Mode only)
🖼️Export Diagram — self-contained interactive HTML with full vector diagram, pan/zoom, and header showing context + date.
🌐Export HTML — full report: summary, interactive diagram, and complete interface list table.
📊Export Excel — interface list as XLSX (Sender · Object · Receiver + extra columns).
🔄New file — returns to the upload screen.
All data is processed locally in your browser — nothing is uploaded or shared.
